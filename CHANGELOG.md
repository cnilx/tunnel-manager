# Changelog

## 1.0.3 - 2026-03-23

### Added
- GUI 顶部标题和托盘提示增加版本号显示。
- 隧道支持 `tags` 标签字段，可在 GUI 中快速筛选。
- 隧道编辑对话框支持复用已有 `SSH主机` 和标签，并允许直接输入。
- 表格右键菜单新增“浏览器中打开”，可在默认浏览器中访问本地端口网页。
- 新增 `CHANGELOG.md` 记录版本变更。

### Changed
- “启动所有隧道 / 停止所有隧道” 现在按当前标签筛选结果生效。
- 添加/编辑隧道对话框布局与尺寸优化，提升可读性和输入体验。
- 表格选中样式弱化，行内操作视觉和点击热区优化。
- “添加隧道”按钮移动到隧道列表标题右侧。
- 窗口右上角关闭按钮默认改为最小化到托盘，而不是弹出关闭方式选择框。
- 配置文件 `source/tunnels.yaml` 调整为本地运行时自动生成，不再作为开发配置提交。

### Fixed
- 持久化 `pid` 后仅凭进程号判断状态的问题，现会校验进程命令行是否匹配当前隧道配置。
- 删除隧道前未先停止对应进程的问题。
- 隧道名称未做唯一性校验导致编辑、删除、PID 写回错位的问题。
- 配置文件并发写入导致的竞态风险，保存流程改为加锁并使用原子替换。
- `remote_host` 为空时会生成无效参数的问题，现会回退到 `127.0.0.1`。
- Tkinter 后台线程直接更新 UI 的线程安全问题。
- 添加/编辑时本地监听端口与现有配置冲突未拦截的问题。
- 启动时本机端口已被占用或与其他隧道冲突时，现在会拦截并弹窗提示原因。
- 支持表格多选后批量删除，`Delete` 键也可触发删除。
- 统一退出清理逻辑，程序异常退出、信号退出和托盘退出时也会尽量保存状态并关闭隧道。
- 修复关闭窗口、右键删除、行内删除与操作列点击热区相关的多处交互问题。

### Notes
- 表格行内操作列当前保留 `启动/停止`、`编辑`、`删除`；`浏览器中打开` 仅保留在右键菜单中。
- 版本发布改为手动执行 `build.bat` 打包，不依赖 GitHub Actions 自动发布。
